ANIME

Anime est une méthode qui permet de créer une animation complexe sur un ou plusieurs éléments du DOM. Anime est capable de gérer les delais et les transitions de manière intuitive.
Sommaire

Syntaxe

anime: [array]

La méthode anime utilise la même syntaxe que la méthode css, à quelques différences près qui sont énoncées ci-dessous.

Spécificitées

Durée et délais

Pour passer d'une transition à une autre anime n'utilise pas de délais comme css mais, des écouteurs d'événements 'transitionend'. Il est donc impératif de vérifier la compatibilité des navigateurs avec les évènements 'transitionend'.

Délais manuels

Les délais manuels ne doivent en aucun cas se situer à la fin d'un objet, comme dans l'exemple suivant :

anime: [°st 1000, {backgroundColor: 'red', delay: 1200}°r /* Le code ci-dessus est incorrect car 'delay' ne doit pas se situer à la fin de l'objet */ °et]

Propriétés non-transitionnables

Il existe de nombreuses propriétés non-transitionnables en css, ces propriétés ne doivent pas (si elles sont seules dans l'objet) être spécifiées avec une durée suppérieure à 0. Sinon l'écouteur attendra indéfiniment qu'une transition se termine, or aucune transition ne se produit puisque la propriété n'est pas transitionnable (c'est le cas de display par exemple).

Voir la liste des propriétés animables

Le code suivant est donc incorrect et peut planter votre script :

anime: [°st 1000, {display: 'none'}°r /* Le code ci-dessus est incorrect car display n'est pas transitionnable */ °et]

Le code correct serait :

anime: [°st 0, {display: 'none'} °et]

Ou avec l'utilisation du mode sans délais :

anime: [°st 1000, {display: 'none'}, 'no-delay'°r /* no-delay permet d'annuler le délais de la transitionDuration */ °et]
